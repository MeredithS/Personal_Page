Projects

Simon!

First project completed using HTML, CSS, and JQuery. The app is a game of short-term memory and response using colors and sounds. Basic features include the user can play against the computer in a series of rounds, each round is more difficult. The app notifies the player whose turn it is and if the player won the round. Special features include the ability to start the game over at any point during the current game and the user cannot interact with the app until the start button is clicked.

Check it out!See the code on github!

WikiNutrition

A Wiki developed using HTML, CSS, JavaScript/JQuery, Ruby, Sinatra, SQL, and ActiveRecord. Basic features include the ability for users to browse articles about nutrition. Users may browse by most recently published or category. If user is signed in then they may make edits to existing articles in markdown or add articles of their own. Advanced features include the ability to add categories from an already existing drop down menu or create new categories that will not be repeated.

Check it out!See the code on github!

Attendance Tracker

A team based project that used the following technologies: Ruby on Rails, ActiveRecord, HTML, CSS, JavaScript, Heroku, Chart.js, Twilio, ActionMailer, and SendGrid. Basic features include permission based accessibility to different features, the ability to send and receive text messages and automatic e-mails. Advanced features are the ability to use information in received text messages to change records and charts with real time information.

Check it out!See the code on github!

Alimenting with Ailments

A single-page application built using a MEAN stack including Mongodb, Express, Angular, and Node.js. Basic features include the ability to create an account, log-in, and view list of foods per ailment. Once logged in user can add ailments to their account. The application will suggest recipes based on the foods that the user is supposed to increase consumption while filtering out recipes that have foods that user should avoid. User has ability to go to the recipe's page or save to favorites.

Check it out!See the code on github!
